笔顺输入法模块

说明:
	首先声明这不是一个输入法，本代码仅完成了输入法词库的生成，及提供一
个简单的例程，其它输入法上层代码需要自己建筑。

	先简单地说明一下如何生成词库及使用例程代码构建自己的输入法：
	1.使用Cygwin或任何其它的类unix系统(32 bit)，执行make命令，将生成
		sorderlib_5000 sorderlib_10000 sorderlib_all
	  三个词库，分别对应5000常用字词库，1w字词库和所有GBK汉字词库
	  (生成时间可以需要几分钟，有个awk程序代码在cygwin下面跑得很慢，偷
	   懒，呵呵)
	2.使用module目录下的示例程序――――非常简单，它提供的最重要函数是
		sorder_query函数
		举个例子，执行sorder_query("123", ...)
		将会返回 横竖撇开始的字符：才本样想权机相楼根格林极核术...
		它们在写入库前就已按照字符使用频率排序，可查看strokeorder.freq.txt

	再简单介绍一下词库结构
		生成几个词库，没有考虑在储存空间上节省，而更多地考虑查询的易用
		性及速度。
		
		库文件分为两部分：笔顺树及数据
		
		笔顺树的结构：
		从文件0字节开始，每24个字节(6个int)为一个树节点，6个int意义：
			{ 当前节点对应的文字数据，
			  '一'对应的结点，
			  '丨'对应的结点,
			  '丿'对应的结点,
			  '丶'对应的结点,
			  '乙'对应的结点
			}
			每个int都是文件位置，读取时只要fseek便可。
			
			文字数据包括
			{ 文字个数N (两字节)
				{ 文字 (两字节) } * N
			}
